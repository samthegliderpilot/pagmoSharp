# Pagmo.NET 1.0.0-beta.1

Release date: `2026-04-18`

## Highlights
- Managed `IProblem` extensibility with stable callback boundary handling.
- Managed `IAlgorithm` extensibility that participates in island/archipelago runtime flows.
- Broad algorithm/problem wrapper coverage with typed log projections and hardened runtime tests.
- Multi-objective helper surface and end-to-end managed multi-objective runtime coverage.
- Self-contained native library on both Windows and Linux: NLopt and IPOPT statically linked — no external solver dependencies required at runtime.

## Behavior Notes
- Release targets: Windows x64 and Linux x64.
- Consumer-facing library/package target: `.NET 8` (consumers on .NET 8, 9, or 10 can reference it).
- Building and running the full test suite on Linux requires the .NET 10 SDK. Tests and examples target `net10.0` internally.
- `wait()` vs `wait_check()` semantics are intentionally distinct (`wait_check()` surfaces deferred runtime errors).
- The native library is built with vcpkg and statically links all dependencies (pagmo2, Boost, TBB, NLopt, IPOPT). On Linux, `libgfortran5` is the only runtime dependency beyond the standard C++ library.

## Optional Solvers
- `nlopt`: **statically linked** — always available in the released build. `OptionalSolverAvailability.IsNloptAvailable` returns `true`.
- `ipopt`: **statically linked** — always available in the released build. `OptionalSolverAvailability.IsIpoptAvailable` returns `true`.
- `snopt7`: not included; supported for users who build from source with their own SNOPT7 license.

## Known Limitations
- Managed thread-clone strategy for non-thread-safe managed problems is deferred.

## Breaking / Compatibility Notes
- None for this first public beta beyond the documented Windows x64/Linux x64 and `.NET 8` release scope.
