# Pagmo.NET 1.0.0-beta.1

Release date: `2026-04-18`

## Highlights
- Managed `IProblem` extensibility with stable callback boundary handling.
- Managed `IAlgorithm` extensibility that participates in island/archipelago runtime flows.
- Broad algorithm/problem wrapper coverage with typed log projections and hardened runtime tests.
- Multi-objective helper surface and end-to-end managed multi-objective runtime coverage.

## Behavior Notes
- Windows-first release target (`x64`).
- Consumer-facing library/package target: `.NET 8`.
- Tests, examples, and documentation tooling may use a newer SDK internally during development.
- `wait()` vs `wait_check()` semantics are intentionally distinct (`wait_check()` surfaces deferred runtime errors).
- Optional solver support is build/environment dependent and validated by feature-gated tests.

## Optional Solvers
- `nlopt`: available when native dependency is present.
- `ipopt`: available when native dependency is present.

## Known Limitations
- Linux/CMake track is deferred post-1.0.
- Managed thread-clone strategy for non-thread-safe managed problems is deferred.

## Breaking / Compatibility Notes
- None for this first public beta beyond the documented Windows x64 and `.NET 8` release scope.
